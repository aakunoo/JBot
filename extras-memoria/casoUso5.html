Caso de Uso: Consultar Clima (UC-05)

ID del Caso de Uso UC-05
Nombre del Caso de Uso Consultar Clima (/clima)
Actor Principal Usuario de Telegram
Descripción Permite al usuario obtener información meteorológica de una localidad específica y, opcionalmente, configurar recordatorios diarios con la previsión para esa localidad. (Funcionalidad aún en desarrollo, susceptible de cambios)
Precondiciones
  • El usuario debe estar registrado (haber utilizado /register previamente).
  • El bot debe estar en funcionamiento y accesible en Telegram.
Postcondiciones
  • El usuario recibe información básica del clima (temperatura, estado, etc.).
  • Si se habilita la función, el bot crea un recordatorio automático para notificar el clima diariamente.
Curso Normal
  1. El usuario envía el comando /clima.
  2. El bot solicita la localidad o provincia para la consulta.
  3. El usuario introduce la localidad (por ejemplo, "Madrid").
  4. El bot obtiene la información meteorológica de una API externa y la muestra al usuario.
  5. Opcionalmente, el bot ofrece programar un recordatorio diario para dicha localidad (ver relación con /recordatorios).
Flujos Alternativos
  • F1: Usuario no registrado
    • El sistema detecta que el usuario no está registrado y muestra un mensaje de error.
    • Se solicita el uso de /register para acceder a la función.
    • Fin del caso de uso.
  • F2: Localidad no reconocida
    • El usuario introduce una localidad no reconocida por la API.
    • El bot muestra un mensaje de error o solicitud de corrección.
Excepciones
  • E1: Error al contactar la API de clima
    • Si la API está caída o hay problemas de conexión, se informa al usuario.
    • El usuario puede reintentar más tarde.
  • E2: Error de conexión con la base de datos (al programar recordatorios)
    • Si falla la inserción del recordatorio, se notifica al usuario y se cancela la programación.
