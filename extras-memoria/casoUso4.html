Caso de Uso: Gestionar Recordatorios (UC-04)

ID del Caso de Uso UC-04
Nombre del Caso de Uso Gestionar Recordatorios
Actor Principal Usuario de Telegram
Descripción Permite al usuario de Telegram crear, editar, eliminar y listar recordatorios dentro del sistema, siempre que se haya registrado previamente mediante el comando /register.
Precondiciones
  • El usuario debe haber usado /register para estar registrado en el sistema.
  • El bot debe estar activo y accesible en Telegram.
Postcondiciones
  • Las operaciones (crear, editar o eliminar) se reflejan inmediatamente en la base de datos.
  • Los recordatorios quedan programados para enviar notificaciones en la fecha y hora especificadas.
Curso Normal
  1. El usuario envía el comando /recordatorios.
  2. El sistema verifica si el usuario está registrado (ver Flujo Alternativo F1 si no lo está).
  3. El sistema muestra un menú con opciones para crear, editar, eliminar o listar recordatorios.
  4. El usuario selecciona la acción deseada e introduce los datos necesarios (título, fecha, etc).
  5. El sistema valida la información y actualiza la base de datos según corresponda.
  6. El sistema confirma la operación y, si es creación o edición, programa el recordatorio.
Flujos Alternativos
  • F1: Usuario no registrado
    • El sistema detecta que el usuario no está registrado y muestra un mensaje de error.
    • El sistema sugiere el uso de /register para acceder a la función.
    • Fin del caso de uso.
  • F2: Datos de recordatorio inválidos
    • Si la fecha u hora introducidas no son válidas, o faltan datos obligatorios, el sistema muestra un error.
    • El usuario puede reintentar con datos correctos.
  • F3: Uso del comando /cancel
    • El usuario escribe en mitad de la creación / edición del recordatorio el comando /cancel.
    • Fin del caso de uso.
Excepciones
  • E1: Error de conexión
    • Si ocurre un fallo de conexión con la base de datos, se informa al usuario del problema.
    • La operación se cancela y el usuario podrá reintentar más tarde.
  • E2: Error al actualizar la base de datos
    • Si no se puede guardar o modificar la información, se muestra un mensaje de error.
    • El usuario puede reintentar la operación o cancelarla.
